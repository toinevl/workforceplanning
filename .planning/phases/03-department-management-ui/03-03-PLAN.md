<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User Action → Delete Mutation | User clicks delete; confirmation dialog shows team count before mutation fires |
| API Response → Delete Dialog | 409 response includes assignedTeamCount; dialog extracts and displays |
| Edit Form → API Patch | Form data serialized and sent to PATCH endpoint; server validates all fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | Injection | DepartmentsSection form submission | mitigate | Form data passed to API via fetchJSON; server validates all fields |
| T-03-14 | Tampering | Delete dialog team count | mitigate | Count sourced from API response (409 body) or live query; user cannot modify |
| T-03-15 | Repudiation | CRUD actions | accept | Single-user app; no audit trail needed |
| T-03-16 | Information Disclosure | 409 error display | mitigate | Only shows team count (integer); no PII exposed |
| T-03-17 | DoS | Delete confirmation spam | accept | UI has loading state; mutations cannot fire twice simultaneously |
| T-03-18 | EOP | Edit department race condition | mitigate | TanStack Query keeps in-memory cache fresh; user selects from current list before edit |
</threat_model>
