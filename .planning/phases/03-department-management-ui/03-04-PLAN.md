<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User Department Dropdown → Migration API | Selected department ID from dialog crosses trust boundary; server validates it exists |
| Migration API Response → UI State | Success/failure response updates button state and shows user feedback |
| Team Form → API Patch | Department assignment in team form; server validates departmentId exists |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-19 | Injection | BulkMigrateButton department selection | mitigate | Department ID sourced from useDepartmentList query (API data); user selects from dropdown |
| T-03-20 | Tampering | Migration confirmation count | mitigate | Count displayed in dialog is from useDepartmentList query (fresh from API); user confirms before proceeding |
| T-03-21 | Repudiation | Bulk migration action | accept | Single-user app; no audit trail needed |
| T-03-22 | Information Disclosure | Migration error messages | mitigate | Only API error text displayed; no system info exposed |
| T-03-23 | DoS | Multiple migration clicks | mitigate | Button disabled after success; mutation prevents simultaneous requests |
| T-03-24 | EOP | Invalid departmentId in dropdown | mitigate | Department ID must exist in useDepartmentList; server validates before update |
</threat_model>
