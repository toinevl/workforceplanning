## Spacing Scale

Declared values (all multiples of 4). Source: existing component patterns in TopNav.tsx,
DepartmentsSection.tsx, settings/page.tsx.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Color badge margin, inline text gaps (`gap-1`) |
| sm | 8px | Compact element spacing, icon-text pairs (`gap-2`, `gap-3`) |
| md | 16px | Default element spacing, card padding (`p-4`), horizontal page padding (`px-4`) |
| lg | 24px | Section padding, vertical page padding (`py-6`) |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Nav link padding is `px-2.5 py-2.5` (10px) — asymmetric hit-target; matches existing Home/Settings nav pattern (source: TopNav.tsx line 53)
- Card inner padding is `p-4` (16px) — matches existing DepartmentCard in DepartmentsSection.tsx
- Stat chip padding is `px-1.5 py-0.5` (6px/2px) — matches version badge in TopNav.tsx line 43

## Color

Source: globals.css, TopNav.tsx, DepartmentsSection.tsx. Background is `#ffffff`. No dark mode
toggle in this phase (dark mode is a CSS media query in globals.css, not a user control).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (`bg-white`) | Page background, card background |
| Secondary (30%) | `#f9fafb` (`bg-gray-50`) | Nav hover state, chip background, skeleton pulse base |
| Accent (10%) | `#111827` (`bg-gray-900` / `text-gray-950`) | Active nav state background (`bg-gray-100`), active nav text (`text-gray-950`), page heading |
| Destructive | `#ef4444` (`text-red-700` / `bg-red-50`) | Error state text and background only |
| Department color | User-defined hex via `dept.color` | Color badge swatch only — applied as `style={{ backgroundColor: dept.color }}` |

Accent reserved for:
1. Active nav link background: `bg-gray-100` when `aria-current="page"`
2. Active nav link text: `text-gray-950` when `aria-current="page"`
3. Page heading `<h1>` text color: `text-gray-900`

Department color swatches are not accent — they are data-driven values from `dept.color` and are scoped exclusively to the 16×16px rounded-full badge element.

Unassigned bucket distinction: border uses `border-dashed` instead of `border-solid`, badge uses `bg-gray-300` (static grey, ignoring `dept.color`).

## Component Inventory

New components for this phase:

### `src/components/departments/DepartmentCard.tsx` (new)

A display-only card (no edit/delete actions — those remain in DepartmentsSection in Settings).

Structure:
```
<div> row container — border border-gray-200 bg-white rounded-lg p-4 flex items-center gap-4
  <span> color badge — h-4 w-4 rounded-full border border-gray-200 style={{ backgroundColor: dept.color }}
  <div> name block — flex-1
    <p> dept name — text-sm font-semibold text-gray-900
  <div> stat chips row — flex items-center gap-2 flex-shrink-0
    <span> headcount chip — text-xs font-medium text-gray-700 bg-gray-50 border border-gray-200 rounded px-1.5 py-0.5
    <span> FTE chip
    <span> team count chip
```

Unassigned variant: replace color badge with a `bg-gray-300` static swatch; apply `border-dashed` to the row container instead of `border-solid`; render name as "Unassigned" in `text-gray-500 italic` (not `text-gray-900 font-semibold`). Detection: `dept.id === 'unassigned'`.

Stat chip label format:
- Headcount: `{n} people` (e.g., "12 people")
- FTE: `{n} FTE` where n = `totalFte.toFixed(1)` (e.g., "8.5 FTE") — always one decimal to prevent float display bugs
- Team count: `{n} team` / `{n} teams` (pluralised) (e.g., "3 teams")

### `src/app/departments/page.tsx` (new)

`'use client'` — uses `useDepartmentList()` TanStack Query hook.
Wraps content in `<AppShell>` with no `board` prop (same as settings/page.tsx pattern).

Layout: `<div className="max-w-6xl mx-auto py-6 px-4">`

Page heading: `<h1 className="text-2xl font-bold text-gray-900">Departments</h1>`
Optional subheading: `<p className="mt-1 text-sm text-gray-600">All departments and rollup stats.</p>`

Card list container: `<div className="mt-6 flex flex-col gap-4">`

### TopNav.tsx (modified)

Add one `<Link>` to the existing `<div className="flex items-center gap-1">` block, between the existing "Settings" link and end of block:

```tsx
const isDepartments = pathname.startsWith('/departments');

<Link
  href="/departments"
  aria-current={isDepartments ? 'page' : undefined}
  className="rounded px-2.5 py-2.5 text-sm font-medium text-gray-700 transition-colors
             hover:bg-gray-50 hover:text-gray-950 focus-visible:outline-none
             focus-visible:ring-2 focus-visible:ring-gray-500
             aria-[current=page]:bg-gray-100 aria-[current=page]:text-gray-950"
>
  Departments
</Link>
```

Active detection: `pathname.startsWith('/departments')` — NOT `pathname === '/departments'`.
This covers both `/departments` (Phase 4) and `/departments/[deptId]` (Phase 5).

## Interaction Contract

### Nav active state

| Trigger | Visual change |
|---------|--------------|
| `pathname.startsWith('/departments')` is true | `bg-gray-100 text-gray-950` on the Departments link |
| Any other path | `text-gray-700` default, `hover:bg-gray-50 hover:text-gray-950` on hover |
| Keyboard focus | `focus-visible:ring-2 focus-visible:ring-gray-500` |

Accessibility: `aria-current="page"` set when active. Screen reader announces current page.

### Loading state (`isLoading`)

3 skeleton cards: `<div className="h-16 w-full animate-pulse rounded-lg bg-gray-100" />`
Source: DepartmentsSection.tsx lines 267-271 (same pattern, same height).

### Error state (`isError`)

```
<div className="rounded-lg border border-red-200 bg-red-50 px-4 py-3">
  <p className="text-sm text-red-700">
    Failed to load departments. {error message}
  </p>
</div>
```

Source: DepartmentsSection.tsx lines 257-262 (identical).

### Empty state (no departments at all, data is `[]`)

```
<p className="mt-6 text-sm text-gray-500">
  No departments yet. Go to Settings to create the first one.
</p>
```

This state occurs only when the API returns an empty array (no real departments AND no unassigned teams). The Unassigned bucket only appears when unassigned teams exist (API contract: appended server-side only when `unassignedTeams.length > 0`).

### DepartmentCard — no hover/click interaction in Phase 4

Cards are display-only in this phase. No link wrapping, no hover lift, no cursor change.
Phase 5 will add the `<Link href="/departments/{dept.id}">` wrapper when the detail page exists.
Exception: Unassigned bucket — never becomes a link (no detail page for `id: 'unassigned'`).
