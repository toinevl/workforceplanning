Create src/lib/api/departments.ts with the following seven functions:

1. **getDepartments()**: Promise<Department[]>
   - Fetch all DepartmentEntity from TABLE_DEPARTMENTS via getTableClient and listEntities
   - Map each entity through entityToDepartment()
   - Sort by sortOrder ascending
   - Return Team[] sorted array

2. **getDepartmentById(id: string)**: Promise<Department | null>
   - Try to fetch DepartmentEntity with partitionKey='department', rowKey=id
   - Return entityToDepartment(entity) on success
   - Return null on not-found (catch and return null, do not throw)

3. **getDepartmentsWithStats()**: Promise<Array<Department & { headcount: number; totalFte: number; teamCount: number }>>
   - This is the critical function for Pitfall 5. Fetch all departments, teams, and members in THREE PARALLEL PARTITION SCANS:
     a) departmentClient.listEntities<DepartmentEntity>() → collect all departments
     b) teamClient.listEntities<TeamEntity>() → collect all teams
     c) staffClient.listEntities<StaffMemberEntity>() → collect all members
   - Use Promise.all() to run three scans in parallel (not sequential)
   - Group in memory:
     - membersMap: Map<baseTeamId, StaffMember[]> to collect members by their base team
     - teamMap: Map<departmentId, TeamEntity[]> to collect teams by department
   - For each department:
     - Get teams assigned to this department via teamMap.get(deptId)
     - For each team, sum up member FTEs and headcounts from membersMap.get(teamId)
     - Add "Unassigned" bucket: teams with no departmentId, compute stats same way
   - Return array of department objects with computed stats, sorted by sortOrder

4. **createDepartment(name: string, color: string, description?: string, deptHead?: string)**: Promise<Department>
   - Generate new uuid for rowKey (use v4())
   - Get current ISO timestamp for createdAt/updatedAt
   - Create DepartmentEntity with:
     - partitionKey: 'department'
     - rowKey: generated uuid
     - name, color, description, deptHead as provided
     - sortOrder: (get highest sortOrder from existing departments + 1, or 1 if none exist) — call getDepartments() to compute
     - createdAt, updatedAt: current timestamp
   - Upsert via departmentClient.upsertEntity(entity, 'Replace')
   - Return entityToDepartment of created entity

5. **updateDepartment(id: string, updates: Partial<{ name: string; color: string; description?: string; deptHead?: string }>)**: Promise<Department>
   - Fetch existing entity via getDepartmentById (error if not found — return null and caller handles 404)
   - Merge updates into entity (do NOT change id, createdAt, sortOrder, partitionKey)
   - Set updatedAt to current ISO timestamp
   - Upsert via departmentClient.upsertEntity(entity, 'Replace')
   - Return entityToDepartment of updated entity

6. **deleteDepartment(id: string)**: Promise<{ deleted: boolean; assignedTeamCount: number }>
   - Query teams table: count all TeamEntity where departmentId === id
   - If count > 0, return { deleted: false, assignedTeamCount: count } (do not delete, caller will return 409)
   - If count === 0:
     - Delete via departmentClient.deleteEntity('department', id)
     - Return { deleted: true, assignedTeamCount: 0 }

7. **bulkAssignUnassignedTeams(defaultDepartmentId: string)**: Promise<{ assigned: number; skipped: number }>
   - Check migration sentinel: fetch { partitionKey: 'migration', rowKey: 'v2-departments' } from TABLE_MIGRATIONS or a hardcoded check
   - Actually, looking at Phase 1 Plan 3, the sentinel was written to TABLE_DEPARTMENTS. Fetch it via departmentClient.getEntity('migration', 'v2-departments'). If not found, this is the first run.
   - Check if sentinel exists; if not, create it now (upsertEntity with current timestamp)
   - Fetch all TeamEntity from TABLE_TEAMS
   - Filter teams with no departmentId (departmentId === undefined)
   - For each unassigned team, call teamClient.upsertEntity({ ...team, departmentId: defaultDepartmentId }, 'Merge')
   - Return { assigned: count of updated teams, skipped: 0 }
   - Pattern: UpdateMode.Merge ensures partial updates don't clobber other fields

Import statement at top:
  - v4 as uuidv4 from 'uuid'

  - type Department, type Team, type StaffMember from '../types/domain'
  - entityToDepartment from '../db/mappers'

Error handling: 
  - use try/catch for getEntity calls (not-found errors)
  - propagate other errors (network, auth)
  
TypeScript: All return types explicit, no implicit any.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API caller → departments.ts domain layer | Untrusted HTTP input will be validated at the route layer (Phase 2 Plan 2); domain layer assumes valid structs |
| departmentId on teams → department assignment | Teams can have optional departmentId; domain functions must handle undefined gracefully |
| Azure Table Storage → in-memory grouping | Three partition scans fetch raw entities; grouping in memory before stats computation reduces round-trips |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | deleteDepartment enforcement | mitigate | Query teams table for departmentId references before deletion; return error response with count if any exist (Phase 2 Plan 2 returns 409) |
| T-02-02 | Information Disclosure | getDepartmentsWithStats aggregation | accept | Rollup stats are computed from public team/member data; no sensitive filtering needed |
| T-02-03 | Denial of Service | getDepartmentsWithStats parallel scans | mitigate | Three partition scans run in parallel (not sequential loops); pagination via listEntities handles large datasets gracefully |
| T-02-04 | Tampering | bulkAssignUnassignedTeams idempotency | mitigate | Sentinel row check prevents duplicate migration runs; UpdateMode.Merge prevents data loss on partial field overwrites (Pitfall ASSIGN-04) |
| T-02-05 | Tampering | createDepartment sortOrder | mitigate | sortOrder is computed deterministically from existing max; not user-provided input |
</threat_model>
