Updated code:
```
export async function GET(req: Request) {
  const teams = await getAllTeams();
  
  // Support optional departmentId query filter
  const { searchParams } = new URL(req.url);
  const departmentId = searchParams.get('departmentId');
  
  if (departmentId) {
    // Filter teams by departmentId (teams with matching departmentId? field)
    const filtered = teams.filter(t => t.departmentId === departmentId);
    return NextResponse.json({ data: filtered });
  }
  
  return NextResponse.json({ data: teams });
}
```

The filtering is straightforward: after fetching all teams (which now include optional departmentId from Phase 1), filter by exact match on departmentId if query param provided. No database-side filtering needed because Teams are already mapped with entityToTeam (which includes departmentId from Phase 1).

Update GET signature to accept req: Request parameter (already available in Next.js route handlers).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → route handlers | Untrusted input from request body and query params; routes validate before calling domain layer |
| Route handlers → domain layer | Trusted: domain functions receive validated inputs only |
| Domain layer → Azure Table Storage | Query results grouped in memory with no user-provided filters; table operations use partitionKey/rowKey constraints |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | POST /departments input validation | mitigate | Validate name and color are non-empty strings at route entry; reject missing fields with 400 |
| T-02-07 | Tampering | PATCH /departments/[id] partial updates | mitigate | Accept only updateable fields (name, color, description, deptHead); block attempts to modify id, createdAt, sortOrder |
| T-02-08 | Tampering | DELETE /departments/[id] cascade | mitigate | Check assigned team count before deletion; return 409 if any teams assigned (prevents orphaned departmentId) |
| T-02-09 | Tampering | POST /admin/migrate-departments authz | accept | No auth layer in v2.0; endpoint is unguarded. Add auth middleware in future milestone. |
| T-02-10 | Information Disclosure | GET /api/teams?departmentId= enumeration | accept | departmentId is public; client can enumerate teams by department (expected behavior for UI filtering) |
| T-02-11 | Denial of Service | GET /api/departments with large result set | mitigate | getDepartmentsWithStats fetches all departments/teams/members; no pagination. For production, add cursor-based pagination in Phase 4 (LIST-01) |
</threat_model>
