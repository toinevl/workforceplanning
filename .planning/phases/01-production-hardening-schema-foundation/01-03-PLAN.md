    Sentinel approach: use `TableClient.upsertEntity()` with `UpdateMode.Replace` and a fixed, deterministic key. Because upsertEntity is an insert-or-overwrite operation, calling it N times with the same key produces exactly one entity — this IS the sentinel pattern.

    Create a helper function `ensureDefaultDepartment(departmentClient: TableClient): Promise<void>` that:
    1. Calls `await departmentClient.upsertEntity({ partitionKey: 'department', rowKey: 'default', name: 'Default', color: '#6b7280', sortOrder: 99, createdAt: new Date().toISOString(), updatedAt: new Date().toISOString() }, 'Replace')`
    2. Returns without any conditional check — upsertEntity guarantees idempotency by design

    Call `await ensureDefaultDepartment(departmentClient)` after the team insertion loop, before member insertion. Import `TableClient` from `@azure/data-tables` and obtain `departmentClient` via `getTableClient(TABLE_DEPARTMENTS)`.
